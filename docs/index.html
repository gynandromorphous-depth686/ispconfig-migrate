What's included

Everything for a clean migration

Each step is a standalone script you can run independently, or chain them together for a full automated migration.

🔍

Server inventory

Run on the old server to auto-detect all vhosts, PHP versions, and DB credentials from WordPress, Joomla, Laravel, and CakePHP config files. Outputs a ready-to-use config.env.

🖥️

Universal installer

Auto-detects your distro and installs Apache, MariaDB, and multi-version PHP using the right package manager.

🔒

Server hardening

CSF firewall (Debian/Ubuntu), firewalld (RHEL/SUSE), Maldet + ClamAV, Fail2ban with ISPConfig log paths.

⚙️

ISPConfig SOAP API

Provisions all your sites via the ISPConfig remote API. Handles clients, PHP-FPM pools, SSL, and subdomain aliases.

🐘

Multi-version PHP

Assigns the correct PHP version (5.6 through 8.4) per site by moving FPM pool configs between PHP versions.

📁

File migration

Rsyncs web files from the old server to ISPConfig document roots with automatic ownership correction.

🗄️

Database migration

Dumps all databases with --single-transaction --routines --triggers --events and imports with user creation.

The scripts

Run them in order

Copy config.env.example to config.env, fill in your server details, then run each script as root on the new server.

Script What it does
collect-inventory.sh new Run on the old server first. Auto-detects all Apache vhosts, document roots, PHP versions, and DB credentials (WordPress, Joomla, Laravel, CakePHP). Outputs config.env with FIXME: placeholders for anything it can't find automatically.
00-install.sh new Universal installer: detects distro, installs LAMP, ISPConfig, CSF/firewalld, Maldet + ClamAV. Supports Debian, Ubuntu, RHEL/AlmaLinux/Rocky, openSUSE.
01-hardening.sh Apache security headers, PHP 7.4 + 8.1 via Sury repo, MariaDB secure install, CSF firewall, Maldet, Fail2ban — for Debian/Ubuntu only.
02-ispconfig-provision.php Creates clients and web domains in ISPConfig via the SOAP remote API.
03-set-php-version.sh Assigns the correct PHP version per site by moving FPM pool configs and updating Apache vhost socket paths.
04-migrate-files.sh Rsyncs web files from old server to ISPConfig document roots with ownership correction.
05-migrate-databases.sh Dumps databases from old server, creates users and databases on new server, imports dumps.
ISPConfig API notes

Quirks we found the hard way

The ISPConfig SOAP API is powerful but has some undocumented behaviours. These took time to track down.

Remote API user is not the admin user

The API user must be created separately in System → Remote Users. The ISPConfig admin login does not work for API calls.

pm is required and must be an enum

The pm field for PHP-FPM must be ondemand, dynamic, or static. Omitting it causes a cryptic validation error.

PHP version is not controlled by the API alone

ISPConfig generates FPM pool configs in the highest installed PHP version's directory. Use 03-set-php-version.sh to move pools after provisioning.

subdomain='www' doubles the prefix

If your domain is already www.example.com, setting subdomain='www' creates a www.www.example.com alias. Set subdomain='none' instead.

CSF moved to a new domain in August 2025

The original ConfigServer developer shut down in August 2025. CSF is now available at https://download.configserver.dev/csf.tgz. The scripts use this new URL.

About

Made by 123Helpdesk

123Helpdesk is a Dutch web hosting provider that has been running LAMP servers since 2008.

These scripts were built during a real-world migration from Ubuntu 18.04 (Apache / MySQL 5.7 / PHP 7.4) to Debian 13 (ISPConfig / MariaDB 11 / PHP 7.4 + 8.1), moving 9 production sites without downtime.

We open-sourced them so others can benefit from the lessons learned — especially around the ISPConfig API quirks that took hours to debug.
